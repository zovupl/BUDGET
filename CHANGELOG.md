# CHANGELOG

## 2026-07-13 — версия 2.1: «Куда ушли деньги» — сворачиваемое окно

Блок «🍩 Куда ушли деньги» на вкладке «Обзор» теперь по умолчанию показывается в
упрощённом виде: маленькое кольцо-диаграмма слева и компактный список категорий с
процентами справа (топ-5, остальные скрыты под «+ ещё N»). По клику блок раскрывается
в прежний подробный вид — большое кольцо, полосы прогресса, число операций, средний
чек и сравнение с прошлым месяцем. Повторный клик по заголовку сворачивает обратно.

## 2026-07-13 — версия 2.0: конверты, подсказки, подробный обзор

**Конверты (бюджеты).** Доходы падают на «Общий счёт», оттуда деньги переводятся
в конверты («Еда», «Хотелки», «Счёт Марины» — создаются вручную). При трате выбирается,
с какого конверта списать; конверт подставляется сам — тот, с которого обычно платят
за эту категорию (запоминается на устройстве). Видно остаток в каждом конверте
и полосу «потрачено из выделенного в этом месяце».
- Новые узлы в базе: `envelopes` (список) и `transfers` (переводы).
- У старых операций конверта нет — считаются как «Общий счёт», ничего не потерялось.
- Удаление конверта возвращает его остаток на Общий счёт, записи остаются в истории.
- Переводы за месяц видны отдельным блоком во вкладке «Лента», их можно удалять.

**Подсказки на «Обзоре».** Приложение само предупреждает: конверт ушёл в минус;
потрачено ≥80% выделенного; одна категория съедает ≥25% всех трат; тратите заметно
больше, чем в прошлом месяце; расходы превысили доходы. И хвалит, когда есть запас.

**Подробный обзор трат.** Под донат-диаграммой — список категорий: сумма в zł, доля в %,
полоса, число операций и средний чек. Внизу — итог за месяц и сравнение с прошлым месяцем.
У доходов тоже появились проценты.

**Круглая ➕ при прокрутке.** Пока ты вверху — обычные кнопки «Трата» и «Доход».
Стоит пролистать вниз, справа появляется круглая ➕: нажатие раскрывает обе кнопки.

Файлы версии: `versions/app_2026-07-13_v2.0.jsx`.

## 2026-07-13 — резервные копии

- `app/scripts/backup.mjs` (`npm run backup`) — скачивает всю базу в `backups/budget_ДАТА.json`,
  хранит последние 30 копий. Пустую базу не сохраняет, чтобы не затереть хорошую копию.
- `app/scripts/restore.mjs` (`npm run restore -- имя_файла.json`) — восстанавливает базу из копии.
  Без аргумента просто показывает, что лежит в самой свежей копии.
- Задание Windows «Budget backup» — копия автоматически каждые 3 дня в 20:00
  (если ПК был выключен, задание отработает при следующем включении).
- Папка `backups/` не уходит на GitHub: в ней фото профилей, а репозиторий публичный.

## 2026-07-12 — версия 1.2

Приложение на экране «Домой» само подхватывает новые версии.

- Раньше iPhone держал сохранённую копию и новую версию не искал — обновления
  были видны только при открытии сайта по ссылке в Safari.
- Теперь при каждом открытии (и при возврате из фона) приложение проверяет обновление
  и, если оно есть, само перезагружается на свежую версию.
- Файлы версии: `versions/main_2026-07-12_v1.2.jsx`, `versions/sw_2026-07-12_v1.2.js`.

## 2026-07-12 — версия 1.0

Первая рабочая версия приложения.

- Прототип `family-budget-v2.jsx` перенесён на Vite + React + Tailwind (папка `app/`).
- Хранилище `window.storage` заменено на **Firebase Realtime Database** (узел `/budget`):
  данные общие для всех устройств, обновляются в реальном времени (`onValue`).
- Записи операций и платежей по долгу пишутся каждая по своему пути — записи с телефона
  Мити и телефона Марины не затирают друг друга.
- Убраны вызовы Anthropic API: разбор фраз («500 кроссовки») делает локальный парсер,
  ответы на вопросы («сколько ушло на еду?») считаются на устройстве, интернет не нужен.
- Чат дополнительно понимает платежи по долгу: «отдал 200 долга» → −200 €.
- Общую сумму долга (3000 €) теперь можно менять прямо в приложении, во вкладке «Ещё».
- Показывается плашка «нет связи», выбранный профиль запоминается на устройстве.
- PWA: manifest, service worker (офлайн-кэш), иконки 💰 на градиенте, полноэкранный режим
  на iPhone при добавлении на экран «Домой».
- Автодеплой на GitHub Pages через GitHub Actions при пуше в `main`.
